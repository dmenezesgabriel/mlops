# Directory Structure

```sh
├── LICENSE                         <- Project license.
├── Makefile                        <- Common commands like `make data`, `make features`, `make train`, `make test`, `make lint`.
├── README.md                       <- Main documentation for setup, usage, development, architecture, and project overview.
├── pyproject.toml                  <- Python project metadata, dependencies, dev tools, build config, and tool settings.
├── uv.lock                         <- Locked dependency versions generated by `uv`; commit this for reproducible environments.
├── .python-version                 <- Python version used by the project, e.g. `3.12`.
├── .gitignore                      <- Files and folders ignored by Git.
│
├── configs                         <- Runtime configuration files for reproducible jobs and pipelines.
│   ├── train.yaml                  <- Training config: dataset paths, features, model params, split strategy.
│   ├── evaluate.yaml               <- Evaluation config: metrics, thresholds, validation datasets.
│   ├── predict.yaml                <- Prediction config: model path, input data, output destination.
│   └── materialize.yaml            <- Feature materialization config: time range, feature views, offline/online targets.
│
├── feature_repo                    <- Feature store repository, e.g. Feast feature definitions and store config.
│   ├── feature_store.yaml          <- Feature store config: registry, provider, offline store, online store.
│   ├── entities.py                 <- Entity definitions used as feature keys, e.g. customer, account, transaction.
│   ├── data_sources.py             <- Batch, stream, or warehouse source definitions for features.
│   ├── feature_views.py            <- Feature view definitions used by training and inference.
│   └── .feastignore                <- Optional ignore file for files not managed by Feast.
│
├── data                            <- Local data storage; usually not committed except small samples or metadata.
│   ├── external                    <- Data from third-party sources.
│   ├── interim                     <- Intermediate transformed data.
│   ├── processed                   <- Final canonical datasets ready for modeling.
│   └── raw                         <- Original immutable data dump.
│
├── docs                            <- Project documentation, architecture notes, data docs, model docs, runbooks.
│
├── models                          <- Local trained models, serialized artifacts, predictions, summaries, and metadata.
│
├── notebooks                       <- Exploratory notebooks and experiments.
│                                  <- Example: `1.0-gdm-initial-data-exploration.ipynb`.
│
├── references                      <- Data dictionaries, manuals, papers, business rules, and explanatory materials.
│
├── reports                         <- Generated reports, analysis outputs, HTML, PDF, Markdown, etc.
│   └── figures                     <- Generated plots, charts, and figures used in reports.
│
├── src                             <- Production source code.
│   └── project_name                <- Main Python package; replace with your real project/package name.
│       ├── __init__.py             <- Makes this directory a Python package.
│       │
│       ├── data                    <- Code to ingest, download, load, validate, and prepare datasets.
│       │
│       ├── features                <- Code that computes/transforms features before registering or materializing them.
│       │
│       ├── models                  <- Code for training, tuning, saving, loading, and managing models.
│       │
│       ├── pipelines               <- Pipeline entrypoints and orchestration logic for automated workflows.
│       │   ├── train.py            <- Training pipeline entrypoint.
│       │   ├── evaluate.py         <- Evaluation pipeline entrypoint.
│       │   └── materialize.py      <- Feature materialization pipeline entrypoint.
│       │
│       ├── evaluation              <- Metrics, validation checks, model comparison, and acceptance criteria.
│       │
│       ├── inference               <- Batch or online prediction logic used after the model is trained.
│       │   ├── batch_predict.py    <- Batch scoring or batch inference entrypoint.
│       │   └── api.py              <- Online inference API entrypoint, e.g. FastAPI.
│       │
│       └── visualization           <- Reusable plotting and visualization code.
│
├── tests                           <- Automated tests.
│   ├── unit                        <- Fast tests for isolated functions/classes.
│   ├── integration                 <- Tests involving multiple modules, files, databases, feature store, or services.
│   └── regression                  <- Tests to prevent old bugs, metric regressions, or output changes.
│
├── deployment                      <- Runtime packaging and deployment files, separated by execution target.
│   ├── training                    <- Deployment files for scheduled or triggered training jobs.
│   │   ├── Dockerfile              <- Container image for training/retraining jobs.
│   │   └── job.yaml                <- Job definition, e.g. Kubernetes Job, Airflow task, Batch job.
│   │
│   ├── inference                   <- Deployment files for prediction runtime.
│   │   ├── Dockerfile              <- Container image for online or batch inference.
│   │   ├── service.yaml            <- Service/deployment definition, e.g. Kubernetes Deployment/Service.
│   │   └── docker-compose.yml      <- Local inference service composition.
│   │
│   └── feature_store               <- Deployment files for feature store materialization or local dependencies.
│       ├── Dockerfile              <- Optional image for feature materialization jobs.
│       └── job.yaml                <- Job definition for feature materialization.
│
└── infra                           <- Infrastructure-as-code or environment provisioning files.
    └── README.md                   <- Notes explaining infrastructure setup, feature store backends, environments, and deployment assumptions.
```
